# A2UI v1.0 协议实现设计文档

> 日期：2026-06-23
> 项目：agent-to-ui (a2a-dev monorepo)
> 状态：已批准

## 一、概述

在 `a2a-dev` monorepo 中实现 A2UI v1.0 协议，包括：

- A2UI 协议核心类型与 Zod v4 验证模式（`@a2a-dev/a2ui-core`）
- 框架无关渲染器核心（`@a2a-dev/a2ui-renderer`）
- A2A 扩展辅助工具（`@a2a-dev/a2ui-extension`）
- Angular v22 前端应用（`apps/a2ui-angular`）
- A2UI 示例智能体服务器（`apps/a2ui-agent`）

## 二、架构方案（方案 C — 混合优化）

参照官方实现模式，结合现有 A2A ExtensionRegistry：

- **服务端**：A2A 服务器通过 AgentCard 声明 A2UI 扩展能力，ExtensionRegistry 用于能力声明
- **智能体**：AgentExecutor 生成 A2UI 消息，打包为 Artifact 中的 DataPart（mimeType: `application/a2ui+json`）
- **传输**：使用 A2A 协议传递，通过 `A2A-Extensions` 头激活
- **客户端**：接收 A2A 响应 → 提取 DataPart → MessageProcessor 解析消息 → SurfaceModel 状态更新 → Angular 渲染

## 五、`@a2a-dev/a2ui-renderer` — 框架无关渲染器核心

### 5.1 架构分层

参照官方 Python SDK `a2ui_core`、TypeScript `web_core` 设计：

| 模块 | 职责 |
|------|------|
| `common/events.ts` | EventSource / EventEmitter 发布订阅系统 |
| `state/component-model.ts` | 单个组件实例（id + type + properties） |
| `state/surface-components-model.ts` | 组件邻接表管理 |
| `state/data-model.ts` | 树状数据模型 + JSON Pointer 路径解析 |
| `state/surface-model.ts` | 单个 surface 状态容器 |
| `state/surface-group-model.ts` | 全局 surface 管理容器 |
| `processing/message-processor.ts` | 消息处理引擎（核心） |
| `catalog/types.ts` | Catalog / ComponentApi 类型 |
| `catalog/function-invoker.ts` | 函数调用器 |

### 5.2 MessageProcessor 核心流程

```
processMessages(messages: A2uiMessage[])
  → for each msg:
    createSurface  → SurfaceGroupModel.addSurface(new SurfaceModel(id, catalogId))
                     + 可选 initialState 中的组件/数据
    updateComponents → surface.componentsModel.addOrUpdate(...)
    updateDataModel  → surface.dataModel.set(path, value)
    deleteSurface    → SurfaceGroupModel.deleteSurface(id)
    actionResponse   → 分发给对应的 action 发起者
    callFunction     → 查找并调用已注册的函数
```

### 5.3 DataModel（JSON Pointer 解析）

- 使用树状结构存储数据
- 支持 `/booking/date` 格式的 JSON Pointer 路径
- 变更时发射 `onUpdated` 事件供 Angular 响应式绑定
- `getSnapshot()` 方法导出完整数据模型

### 6.3 Text 组件 Markdown 逻辑（参照官方 Angular 实现）

- **非 Markdown 变体**：`h1`-`h5`、`caption` → 直接使用 `<h1>` 等标签
- **Markdown 变体**：`body`、`span` → `[innerHTML]` 绑定
- **异步渲染**：`effect()` 监听 text 变化 → 异步调用 markdown-it → 更新 signal
- **竞态处理**：`renderRequestId` 递增标记防竞态
- **样式**：`::ng-deep` 穿透 innerHTML

### 7.2 ComponentHost 动态组件渲染

核心机制：通过 Angular `ngComponentOutlet` 根据组件类型动态解析并渲染对应的 Angular 组件。

```typescript
@Component({
  selector: 'a2ui-component-host',
  template: `
    @let comp = resolvedComponent();
    @if (comp) {
      <ng-container *ngComponentOutlet="comp" />
    }
  `,
})
export class ComponentHostComponent {
  surfaceId = input.required<string>();
  componentKey = input.required<{ id: string; basePath: string }>();

  resolvedComponent = computed(() => {
    const surface = this.renderer.getSurface(this.surfaceId());
    const model = surface?.componentsModel.get(this.componentKey().id);
    if (!model) return null;
    return this.catalog.getAngularComponent(model.type);
  });
}
```

## 八、A2A 传输整合

### 8.1 数据流

```
AgentExecutor
  ├─ 生成 A2UI 消息数组 [{createSurface:...}, {updateComponents:...}]
  └─ 打包为 Artifact:
       artifact.parts[0] = {
         data: { a2uiMessages: [...] },
         mediaType: "application/a2ui+json"
       }
       ↓
A2A 传输
       ↓
A2AClient → 收到 Task → extractA2uiFromTask()
       ↓
A2uiRendererService.processMessages(a2uiMessages)
       ↓
SurfaceGroupModel 更新 → Angular 渲染
```

### 8.2 AgentCard 扩展声明

```typescript
capabilities: {
  extensions: [{
    uri: "https://a2ui.org/a2a-extension/a2ui/v1.0",
    description: "A2UI 协议 — 代理驱动的声明式 UI",
    required: false,
  }],
}
```
