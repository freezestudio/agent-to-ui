# A2UI 基本目录实现指南

本指南专为实现 A2UI 基本目录（v1.0）的渲染器和客户端开发者设计。它详细说明了如何视觉呈现和功能实现目录中定义的每个组件和客户端函数。

在通用 A2UI 绑定之上构建框架特定适配器时，请参考本文档以获取预期的视觉行为、建议的布局和交互模式。本指南使用了适用于 Web、移动端（iOS/Android）和桌面平台的通用术语。

---

## 目录的强制标识符规则

A2UI v1.0 目录中定义的所有实体——包括**组件类型**、**函数名称**和**参数/属性名称**——必须严格遵守 [Unicode 标准附录 #31 (UAX #31)](https://www.unicode.org/reports/tr31/) 的变量命名规则：

- 必须以 `XID_Start` 或下划线（`_`）开头。不能以数字（`Nd`）开头。
- 后续字符必须属于 `XID_Continue`。
- 必须严格排除符号和空白（`Pattern_Syntax` 和 `Pattern_White_Space`）。

规范正则表达式验证：`^[\p{XID_Start}_][\p{XID_Continue}]*$`

## 1. 组件

基本目录包含以下组件：

- **Text**：显示文本内容。应尽可能使用 Markdown 解析器。
- **Image**：从 URL 显示图像。
- **Icon**：显示标准系统图标。
- **Video**：视频播放器。
- **AudioPlayer**：音频播放器。
- **Row**：水平布局容器。
- **Column**：垂直布局容器。
- **List**：可滚动的组件列表。
- **Card**：具有卡片样式（圆角、阴影）的容器。
- **Tabs**：一组标签页，每个都有标题和对应的子组件。
- **Divider**：分隔线。
- **Modal**：对话框窗口。
- **Button**：分发协议操作的交互式按钮。
- **TextField**：用户文本输入字段。
- **CheckBox**：带有标签的可切换控件。
- **ChoicePicker**：从列表中选择一个或多个选项的组件。
- **Slider**：在范围内选择数值的控件。
- **DateTimeInput**：日期和/或时间输入。

有关完整实现细节，请参阅原始英文文档。
