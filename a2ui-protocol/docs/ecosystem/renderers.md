# 生态系统渲染器

社区和第三方 A2UI 渲染器实现。

> **注意**
> 这些渲染器由其各自作者维护，而非 A2UI 团队。请检查每个项目的兼容性、版本支持和维护状态。

> **提示**
> 正在寻找**官方** A2UI React 渲染器？参见 [`@a2ui/react`](https://www.npmjs.com/package/@a2ui/react) — A2UI 团队维护的核心 A2UI React 渲染器。

### 亮点

**easyops-cn/a2ui-sdk** (`@a2ui-sdk/react`) 是功能最完整的社区 React 渲染器，有 11 个已发布版本、Radix UI 原语、Tailwind CSS 样式和专用文档站点。在 [A2UI discussions](https://github.com/a2ui-project/a2ui/discussions/489) 上宣布。对于官方 A2UI React 渲染器，参见 [`@a2ui/react`](https://www.npmjs.com/package/@a2ui/react)。

**lmee/A2UI-Android** 填补了重要空白——它是目前唯一的 Jetpack Compose 渲染器，覆盖 Android 5.0+，包含 20+ 组件、数据绑定和无障碍支持。

**sivamrudram-eng/a2ui-react-native** 是唯一的 React Native 渲染器，通过单一代码库在 iOS 和 Android 上启用 A2UI。

**BBC6BAE9/a2ui-swift**（原名 **a2ui-swiftui**）是一个原生 SwiftUI 渲染器，支持 iOS、macOS、visionOS、watchOS 和 tvOS。覆盖所有 18 个基本的 A2UI 组件，支持双 v0.8 + v0.9 协议，包含 JSONL 流式传输，并使用 Observation 框架实现响应式。

**TanXudong-Vivo/A2UI-Android-Renderer** 是一个模块化的 Android 渲染器，使用 Jetpack Compose 和 Material 3 构建，是第一个支持 A2UI v0.9 协议的 Android 实现。覆盖 13 个完全实现的组件（包括通过 Coil 加载图像）、来自 LLM 令牌流的流式渲染、基于 `path` 表达式的数据绑定和 `formatDate`，以及用于注册额外组件类型的可插拔自定义目录。演示应用包括与官方 `restaurant_finder` ADK 代理的实时连接。

**AGenUI/AGenUI** 是一个跨平台的原生 A2UI v0.9 渲染器，通过共享的 C++ 核心支持 iOS、Android 和 HarmonyOS。专为高性能、可扩展性和跨平台一致性而设计。AGenUI 完全实现了 A2UI v0.9，并提供用于扩展 UI 组件和函数调用的运行时 API。它还引入了[自定义目录](https://github.com/AGenUI/AGenUI/blob/main/agenui_catalog.json)，用 Table、Carousel、Web、RichText 和用于更丰富外观和布局控制的 Styles 属性扩展了基本目录，同时与 A2UI 的可扩展目录模型保持一致。参见[组件演示](https://genui.amap.com/components)、相应的开源 [A2UI 生成技能](https://github.com/AGenUI/AGenUI/tree/main/skills/a2ui-generation)，并在 [GitHub](https://github.com/AGenUI/AGenUI) 上了解更多。

**lynx-family/lynx-stack** (`@lynx-js/genui/a2ui`) 为 A2UI v0.9 提供了 ReactLynx 渲染器。它通过 `MessageStore` 消费经过验证的服务器到客户端 A2UI 消息，从调用者提供的目录中渲染已批准的 ReactLynx 组件，并通过 `onAction` 转发生成的 UI 操作。发布的 `@lynx-js/genui` 包通过 `a2ui` 子路径暴露 A2UI 渲染器，而 GenUI CLI 支持构建时目录产物和 A2UI 系统提示生成。

## 提交渲染器

如果你构建了 A2UI 渲染器，请提交以在此列出。

### 如何提交

要提交渲染器，请遵循以下步骤：

1. **Fork** [a2ui-project/a2ui](https://github.com/a2ui-project/a2ui) 仓库
2. **编辑**此文件（`docs/ecosystem/renderers.md`）— 在社区渲染器表中添加一行，包含你的渲染器名称、平台、npm 包（如有）、版本支持和源代码链接
3. **开启 PR** 到 `a2ui-project/a2ui`，附带渲染器的简短描述
4. **在 [GitHub Discussions](https://github.com/a2ui-project/a2ui/discussions) 中发帖** — 让社区知道你构建了什么！一个简短的演示视频非常有帮助。

需要灵感？浏览仓库中的 **[社区示例](https://github.com/a2ui-project/a2ui/tree/main/samples)**——这些覆盖了 Angular、Lit 和基于 ADK 的代理，是一个很好的起点。

### 好的社区渲染器的特点

如果满足以下条件，列表更有可能被接受和使用：

- 有**已发布的源代码**（首选开源，MIT 或 Apache 2.0）。
- 清楚地说明**支持哪个 A2UI 规范版本**（v0.8、v0.9 或两者）。
- 覆盖其他 A2UI 渲染器的**基本组件**：文本、按钮、输入框、基本布局组件等。
- 包含带有安装说明和最小使用示例的 **README**。
- **积极维护**——如果你不再支持，请将其标记为已归档。

社区渲染器不需要达到生产就绪才能被列出——实验性和早期阶段的项目欢迎进入"值得注意的项目"部分。
