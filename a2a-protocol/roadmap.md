# A2A 协议路线图

**最后更新：** 2026年3月10日

## 近期计划

- 发布协议的 `1.0` 版本，这代表了协议的重要成熟，具有增强的清晰度、更强的规范和重要的结构改进。
- [A2A 协议 v1.0 新特性](https://a2a-protocol.org/latest/whats-new-v1/)有进一步的更新。
- 继续支持额外的 [A2A 扩展](topics/extensions.md) 并提供 SDK 支持。
- 优先考虑社区主导的开发，通过标准化流程为规范、SDK 和工具做出贡献。

要查看最近的协议变更，请参阅[发布说明](https://github.com/a2aproject/A2A/releases)。

## 长期路线图（3-6个月）

### 治理

TSC 寻求通过 [A2A 扩展](topics/extensions.md)、[示例](https://github.com/a2aproject/a2a-samples)和参与来简化贡献机会。

### 验证

随着 A2A 生态系统的成熟，A2A 社区拥有工具来验证其智能体变得至关重要。社区已启动两项工作来帮助验证。了解更多关于 [A2A Inspector](https://github.com/a2aproject/a2a-inspector) 和 [A2A 协议技术兼容性工具包](https://github.com/a2aproject/a2a-tck)（TCK）的信息。

### SDK

A2A 项目目前托管六种语言的 SDK（Python、Go、JS、Java、.NET 和 Rust）。

### 社区最佳实践

随着公司和个人以越来越快的速度部署 A2A 系统，我们希望通过收集和分享 A2A 支持的最佳实践和成功故事来加速社区的学习。
